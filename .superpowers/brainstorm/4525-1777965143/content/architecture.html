Data Flow — One Game Round
Player submits card → POST /api/game/play → Redis: store card in round → Pub/Sub: broadcast "card played" → WS handler pushes event to all clients
Czar picks winner → POST /api/game/pick → Redis: update scores → PostgreSQL: save round result → Pub/Sub: broadcast "round ended"
Key Modules

🃏 Card Seeder

One-time script: fetches all 73 packs from REST Against Humanity API → stores in PostgreSQL. Never hits external API at runtime.

🎮 Game Engine

Server-side state machine: WAITING → DEALING → PLAYING → JUDGING → SCORING → ENDED. Lives in a shared server module.

📡 WS Hub

Vinxi WebSocket route per game room. Subscribes to Redis channel for that room, pushes all events to connected clients.

🏠 Lobby

Host creates room → 6-char code. Players join by code + name. Host configures packs, rounds, max players, house rules.
